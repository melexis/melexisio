Param rules:

  • sa = 7-bit I2C address (hex, no 0x). Validation via ReadParameterU8x.
  • Arguments after first ':' become comma-delimited before handler sees them.

2. Measurement Formats

RAW: raw:66:<tick_hex>:<max_raw_hex>,<p0>,...,<p767> MV: mv:66:<tick_dec>:<min_to>,<t0>,...,<t767> Values:

  • tick: osKernelSysTick()
  • max_raw: 4 hex digits
  • temperatures: 2 decimals Errors: raw:66:ERR:<status> mv:66:ERR:<status>

3. Mode Control

; -> queue STATE_WEB_CONTINUOUS ! -> queue STATE_WEB_INTERACTIVE Handler prints status line then suppresses prompt (no_result=1).

5. Error Handling

  • Parameter parsing failures set hterm->result != HAL_OK (generic error; most handlers do not emit a specific error line yet).
  • RAW/MV emit explicit ERR code lines on MLX read failure. Improvement: adopt uniform err:<cmd>:<code>:<detail> pattern.

6. Concurrency

RAW and MV protected by webMutexHandle during MLX90642 frame read.

7. Implementation Notes

  • Token uppercased via strupr() before matching.
  • After command match, ':' in remainder replaced with ',' enabling simple comma token parsing inside handlers.
  • Prompt suppressed when no_result=1.
